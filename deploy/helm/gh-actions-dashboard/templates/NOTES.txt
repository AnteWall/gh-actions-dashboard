=== GETTING STARTED ===

{{- if .Values.dashboard.enabled }}

1. Get the Dashboard URL:
{{- if .Values.ingress.enabled }}
{{- range $host := .Values.ingress.hosts }}
  {{- range .paths }}
   http{{ if $.Values.ingress.tls }}s{{ end }}://{{ $host.host }}{{ .path }}
  {{- end }}
{{- end }}
{{- else if contains "NodePort" .Values.dashboard.service.type }}
  export NODE_PORT=$(kubectl get --namespace {{ .Release.Namespace }} -o jsonpath="{.spec.ports[0].nodePort}" services {{ include "gh-actions-dashboard.dashboard.fullname" . }})
  export NODE_IP=$(kubectl get nodes --namespace {{ .Release.Namespace }} -o jsonpath="{.items[0].status.addresses[0].address}")
  echo http://$NODE_IP:$NODE_PORT
{{- else if contains "LoadBalancer" .Values.dashboard.service.type }}
     NOTE: It may take a few minutes for the LoadBalancer IP to be available.
           You can watch the status by running:
           kubectl get --namespace {{ .Release.Namespace }} svc -w {{ include "gh-actions-dashboard.dashboard.fullname" . }}
  export SERVICE_IP=$(kubectl get svc --namespace {{ .Release.Namespace }} {{ include "gh-actions-dashboard.dashboard.fullname" . }} --template "{{"{{ range (index .status.loadBalancer.ingress 0) }}{{.}}{{ end }}"}}")
  echo http://$SERVICE_IP:{{ .Values.dashboard.service.port }}
{{- else if contains "ClusterIP" .Values.dashboard.service.type }}
  kubectl port-forward --namespace {{ .Release.Namespace }} svc/{{ include "gh-actions-dashboard.dashboard.fullname" . }} 3000:{{ .Values.dashboard.service.port }}
  Then visit: http://localhost:3000
{{- end }}
{{- end }}

=== CONVEX BACKEND ===

The Convex backend is running at:
  Internal: {{ include "gh-actions-dashboard.convex.internalUrl" . }}
  HTTP Actions: {{ include "gh-actions-dashboard.convex.httpActionsUrl" . }}

{{- if .Values.convex.deploy.enabled }}
Convex functions are being automatically deployed via a Helm hook job.
Check the deployment job status:
  kubectl get jobs --namespace {{ .Release.Namespace }} -l "app.kubernetes.io/instance={{ .Release.Name }}"

View deployment logs:
  kubectl logs --namespace {{ .Release.Namespace }} -l "app.kubernetes.io/component=convex-deploy,app.kubernetes.io/instance={{ .Release.Name }}"

{{- if not .Values.secrets.convexAdminKey }}
⚠️  WARNING: convex.deploy.enabled is true but secrets.convexAdminKey is not set!
   The Convex function deployment will fail.

   To get an admin key, first wait for Convex to start, then run:
   kubectl exec --namespace {{ .Release.Namespace }} -it $(kubectl get pod --namespace {{ .Release.Namespace }} -l "app.kubernetes.io/component=convex,app.kubernetes.io/instance={{ .Release.Name }}" -o jsonpath="{.items[0].metadata.name}") -- ./generate_admin_key.sh

   Then upgrade the release with the admin key:
   helm upgrade {{ .Release.Name }} <chart> --set secrets.convexAdminKey=<your-key>
{{- end }}
{{- else }}
To manually deploy Convex functions:
1. Port-forward the Convex API:
   kubectl port-forward --namespace {{ .Release.Namespace }} svc/{{ include "gh-actions-dashboard.convex.fullname" . }} 3210:{{ .Values.convex.service.apiPort }}

2. Set environment variables and deploy:
   export CONVEX_SELF_HOSTED_URL=http://localhost:3210
   export CONVEX_SELF_HOSTED_ADMIN_KEY=<your-admin-key>
   npx convex deploy
{{- end }}
{{- end }}
